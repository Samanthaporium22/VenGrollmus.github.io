Project Idea

Overview

The project is to develop a mobile Customer Relationship Management (CRM) application which will synchronise with a desktop/web application. This will allow the sales and technical team maintain customer and supplier details while on the road interacting with the customers and suppliers. The technical team may also have jobs in the field that they need update records on; and rather than keeping paperwork and notes, and then entering this information into the system when they return to the office, the application will provide this. This will then free up time for the team, allowing them to focus on billable work and thus increasing revenue for the business.

Motivation

Working in a small business, where funds are extremely tight, the majority of CRM’s available that are useful for them are not free, and have a monthly fee attached to them. The aim of this application is that it will be a pay once, and they own the application. It will be their responsibility to host the application on their own desktop/webserver to ensure that the data is synchronised. Utilising Visual Studio, a solution will be easily developed to solve a problem for small businesses to maintain a CRM.

Description

The application will allow the user to do the following: -

  • Create, update, and delete Customer or Supplier details.
  • Manage notes.
  • Send Emails.
  • Manage Invoices.
  • Manage Purchase Orders.

The users will be able to track all touch points with the clients or suppliers, tracking of orders that the clients place as well as orders that they place with their suppliers. The application will also track projects that the client engages the user, so that the user can track estimated effort for the project against the actual effort, as well as the estimated revenue against actual revenue. This can also be applied against suppliers when they are engaged by the user to provide support / services for a project.

All this information will provide the user with details as to who their best and worst performing suppliers are in relation to providing services and products compared to each other. The user will also be able to see who their most valuable clients are, in relation to revenue and touch points against those that may be high maintenance and low profitable.

Tools and Technologies

The tools, technologies, and hardware that this project will need will be Visual Studio to enable the coding of the Mobile Application, Website and interfaces between the mobile application and the website. GitHub will be used as the source control tool to ensure that the code is in a centralised repository for the team. The main communication tool will be Microsoft Teams for us to hold regular meetings, which we can also use to record the meetings as well. Microsoft Word will be utilised to document the meeting minutes and discussions. Trello can be used to ensure that we understand where we are with our tasks and deliverables.

Skills Required

The skills for this project at a minimum would be a basic understanding of HTML, Database Interaction, App Development, and the ability to create API interaction between a mobile app and a website. The tools needed will be Visual Studio for the coding, and access to GitHub for source control. I feel with the current people undertaking the course, we will be able to find team members to cover the skills needed. This will ensure that those that may lack some of the skills will be able to improve their skill set, as well as imparting their knowledge to others.

Outcome

A successful outcome of the project will allow users to have access to their customer and supplier details while on the road and be able to update customer and supplier details and information surrounding the details. This will then remove the need for the user to makes notes using pen and paper and have the details accessible to staff who are back in the office. The development of this project could be impacted by the ability of the various team members to access a centralised database to communicate with, and this will need to be considered.
